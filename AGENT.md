# AGENT

## 작업 로그 규칙 (mandatory)
- 모든 작업 완료 또는 중단 시점마다 `HANDOFF.md`에 기록한다.
- 기록은 최신 순으로 맨 위에 추가한다.
- 각 항목은 최소 다음 항목을 포함한다.
  - 실행 작업 요약(무엇을, 왜 했는지)
  - 변경 파일 목록
  - 현재 상태(완료/진행/차단)
  - 다음 액션 또는 리스크

## `HANDOFF.md` 기록 형식
- 제목: 날짜 또는 세션 단위 섹션
- 하위 항목: 진행 요약, 이번 작업 산출물, 남은 항목, 참고 상태

## 원격 저장소 보안 규칙 (mandatory)
- `git`/`gh`/`rebase`/`revert`/`push`/`pull`/`merge`/`commit` 관련 행위에서는 다음 항목을 절대 포함하지 않는다.
  - 액세스 토큰, API 키, 비밀키, 세션 ID, 인증 헤더 값
  - 이메일 주소, 실제 인적 식별 정보, 실명/닉네임/아이디/비밀번호
  - 전화번호, 카드 정보, 주소, 신분증/민감 식별자
- 위 항목이 포함되었을 가능성이 있으면 해당 동작을 중단하고 값을 치환/삭제한 뒤 재시도한다.
- 커밋 메시지는 기능/작업 중심으로 작성하고, 개인식별값·민감정보를 넣지 않는다.
- 파일명, 브랜치명, PR 제목/본문에도 민감값을 넣지 않는다.

### 커밋 전 점검(권장)
- 토큰/비밀 추정 패턴 검사
  - `rg -n --hidden -S "ghp_[A-Za-z0-9_]{20,}|AKIA[0-9A-Z]{16}|xox[baprs]-[0-9A-Za-z-]+|password|passwd|api[_-]?key|bearer" .`
- 이메일 패턴 검사
  - `rg -n --hidden -S "[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}" .`
- 개인식별 토큰성 텍스트가 있으면 즉시 마스킹/삭제 후 커밋한다.

## 예외/주의
- `HANDOFF.md`는 추적용 문서이며 기능 코드와 동일한 신뢰도로 해석한다.
- 수동 QA 완료 후 결과는 `QA_RESULTS.md`에 반영한다.
- `HANDOFF.md`는 운영 추적 문서로, 보존이 필요할 수 있으므로 `.gitignore`로 제외 처리한다.
