# QA CHECKLIST (Day 6~7)

## 핵심 범위
- MVP 단일 루프: `이벤트 발생 -> 선택 -> 결과 처리 -> 피드백 -> 다음 라운드`
- 오프라인 저장/복구 최소 기준
- UX 핵심: 라운드 선택, 결과 피드백, XP/연속 출근 노출

## T10: 저장/복구 수동 시나리오

1. 새 게임 시작
- [ ] 앱 실행 시 `GameState.initial` 상태로 시작하는지
- [ ] `READY -> CHOICE` 전이가 자동으로 완료되는지

2. 라운드 1 완료 후 저장
- [ ] 라운드 결과 표시 후 `POST_REVIEW` 진입 확인
- [ ] 종료 시 `ISessionRepository.save` 호출로 상태가 저장되는지

3. 앱 재시작 시 복구
- [ ] 재실행 직후 동일 `playerId` 로 마지막 상태가 로드되는지
- [ ] `POST_REVIEW` 상태였다면 화면에서 결과/미션 완료 상태가 유지되는지
- [ ] `DAY`, `cash`, `xp`, `streak`, `flowState` 값이 일치하는지

4. 아카이브
- [ ] `archive(playerId)` 호출 시 캐시에서 삭제되는지
- [ ] 삭제 후 새 게임 시작으로 분기되는지

## T11: 라운드 핵심 경로 통합

1. 라운드 엔진 경로
- [ ] `startRound()` 이후 `CHOICE` 노출
- [ ] `choose()`가 유효하지 않은 ID에 대해 에러 처리
- [ ] `resolve()`에서 `SIMULATE` 선행 조건 검증
- [ ] `resolve()` 성공 시 `POST_REVIEW` 전이
- [ ] `proceedNextRound()` 후 `CHOICE` 전이(다음 DAY)

2. 보상 경로
- [ ] 첫 선택 미션(`first_choice`) 시 XP 보너스 반영(+5)
- [ ] 미션 완료 후 다음 라운드에서 미션이 리셋되는지
- [ ] 음수 수익 시 연속 출근이 0으로 리셋되는지

3. UI 연결
- [ ] CHOICE에서 옵션 목록 렌더
- [ ] 선택 시 결과 패널 표시
- [ ] 결과 패널에서 다음 라운드 버튼 동작

## T12: 기본 품질 점검 (기능 우선 범위)

- [ ] 앱 시작~1라운드 완료까지 프레임 드롭이 없고 오류 없이 렌더되는지
- [ ] 라운드별 소요 시간 90초 내 종료 가능한지(목표)
- [ ] 화면 상단의 `최근 라운드` 표시가 라운드별 소요 시간(ms)로 갱신되는지
- [ ] 예외 메시지 노출이 일관적인지 (무효 선택, 상태 오류)
- [ ] `analyticsSink` 이벤트/에러 트래킹 호출이 1회 라운드당 적절히 발생하는지
- [ ] 미션/XP/연속 출근 텍스트가 화면 하이라이트에 노출되는지
